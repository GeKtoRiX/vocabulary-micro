# AGENTS.md

## Назначение
Этот файл является главным policy-документом для агентной работы в репозитории. Его нельзя менять без прямого запроса пользователя.

## Evolution Cycle 1
Текущий цикл управления: `1`.

## Базовые правила
1. Перед любой mutating-задачей прочитать `AGENTS.md`, `MEMORY.md`, `CONTINUITY.md` и `docs/LLM_PROJECT_MAP.md`.
2. Сначала подтвердить текущее состояние репозитория через чтение кода и документации, затем менять файлы.
3. После завершённого mutating-шага или явной контрольной точки обновить `CONTINUITY.md`.
4. Если в ходе работы принято архитектурное или процессное решение, обновить `MEMORY.md`.
5. Если команда пользователя противоречит этому файлу или зафиксированным решениям из `MEMORY.md`, остановить выполнение, описать конфликт и предложить безопасный путь.

## Язык
- Ответы агента писать на русском языке.
- Новые комментарии, добавленные агентом в код, писать на русском языке.
- Существующие английские docstring и комментарии массово не переводить.
- При точечном редактировании сохранять локальную консистентность файла, если перевод не нужен для задачи.

## Безопасность
- Реальные секреты не коммитить.
- Секреты хранить только в `.env` и `.env.*.local`.
- Шаблоны и примеры хранить в `.env.example`, `.env.postgres.example`, `.env.compose.*.example`.

## Рабочий цикл
`read -> do -> log -> save`

- `read`: изучить релевантные файлы и текущее состояние.
- `do`: выполнить изменение с минимальным затрагиванием прикладного поведения.
- `log`: записать значимые решения в `MEMORY.md`.
- `save`: обновить `CONTINUITY.md` так, чтобы следующая сессия могла продолжиться без догадок.

## Память проекта
### MEMORY.md
- Используется как долговременная память проекта.
- Обновляется только при архитектурных, инфраструктурных или процессных решениях.
- Записи должны содержать дату, действие и краткое обоснование.

### CONTINUITY.md
- Используется как краткая точка восстановления сессии.
- Обязательные поля: `Current Task`, `Progress`, `Blocked by`, `Next Step`, `Last Updated`.
- `Next Step` должен содержать одну конкретную инструкцию для продолжения работы.

## Файлы автозагрузки
- `AGENTS.md` — автозагружается агентами OpenAI Codex.
- `CLAUDE.md` — автозагружается агентами Claude Code (Anthropic).
- Оба файла описывают одни и те же правила работы; `CLAUDE.md` дополнительно содержит quick reference по инструментам.
- При изменении policy синхронизировать оба файла.

## Границы bootstrap-фазы
- Не менять публичные интерфейсы `core` без отдельного запроса.
- Не менять runtime-логику parse/sync, HTTP gateway и storage-адаптеров в рамках bootstrap-задач governance.
- Не трогать незакоммиченные пользовательские изменения, не относящиеся к текущей задаче.
